# Story 5.2: 迭代上限处理

Status: review

## Story

As a 系统架构师,
I want 迭代达到上限时转人工处理,
so that 可以避免无限循环.

## Acceptance Criteria

1. **AC1: 上限检查**
   - Given 迭代次数 = max_iterations
   - When Evaluator 再次返回失败
   - Then 任务状态流转到 "待人工处理"
   - And 系统提示用户迭代已达上限

2. **AC2: 建议提示**
   - Given 达到迭代上限
   - When 转人工处理
   - Then 建议用户检查需求理解是否正确

## Dev Notes

### 实现说明

**已实现部分** (`internal/harness/iteration.go`):
- `CheckLimit(taskID)` 方法检查是否达到上限
- `ShouldContinue(taskID, evaluatorOutput)` 判断是否继续迭代
- 达到上限时 Orchestrator 状态变为 "needs_attention"

### 上限处理流程

```
迭代失败 → ShouldContinue → 检查上限
                ↓
         达到上限? → 状态 = needs_attention
                ↓
         未达上限? → 继续迭代
```
