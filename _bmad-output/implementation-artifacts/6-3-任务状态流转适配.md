# Story 6.3: 任务状态流转适配

Status: review

## Story

As a 系统架构师,
I want 任务状态正确流转,
so that 可以追踪整体进度.

## Acceptance Criteria

1. **AC1: Planner 完成流转**
   - Given Planner 完成
   - When 所有 P1-P5 完成
   - Then 父任务状态更新为 "Planner 完成"
   - And Generator 开始执行

2. **AC2: Generator 完成流转**
   - Given Generator 完成
   - When G4 完成
   - Then 父任务状态更新为 "Generator 完成"
   - And Evaluator 开始执行

3. **AC3: Evaluator 完成流转**
   - Given Evaluator 完成且通过
   - When 所有 E 项通过
   - Then 父任务状态更新为 "完成"
   - And 触发 Git 提交

4. **AC4: 失败流转**
   - Given Evaluator 完成且有失败
   - When 存在失败项
   - Then 触发迭代修复
   - Or 达到上限转人工处理
