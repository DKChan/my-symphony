# Story 7.3: 失败报告展示

Status: review

## Story

As a 开发者,
I want 在 Web 页面查看失败报告,
so that 我可以了解失败原因.

## Acceptance Criteria

1. **AC1: 失败报告展示**
   - Given 任务迭代失败
   - When 用户点击失败详情
   - Then 显示完整失败报告
   - And 显示失败的测试用例
   - And 显示代码审计问题
   - And 显示风格评审问题

## Dev Notes

### 实现说明

**已实现部分** (`internal/harness/evaluator.go`):
- `EvaluatorOutput.FailureReport` 存储失败报告
- `generateFailureReport()` 生成 Markdown 格式报告
- 报告包含 BDD/TDD 测试失败、代码审计问题、风格问题

### 失败报告格式

```markdown
# 评估失败报告

## BDD 测试失败
- 通过: 3/5
- 失败用例: Test1, Test2

## TDD 测试失败
- 通过: 2/3
- 失败用例: TestX

## 代码审计问题
- Missing error handling

## 代码风格问题
- Inconsistent naming
```

### Web 展示

前端可通过 EvaluatorOutput.FailureReport 获取 Markdown 格式的失败报告，使用 Markdown 渲染展示。
