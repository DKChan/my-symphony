# Story 2.2: 需求澄清 (人工参与)

Status: review

## Story

As a 开发者,
I want 系统调用 BMAD PM Agent 进行需求澄清,
so that AI 能理解我的需求意图.

## Acceptance Criteria

1. **AC1: 需求澄清调用**
   - Given 一个新任务进入 Planner 阶段
   - When 阶段开始执行
   - Then 系统调用 BMAD PM Agent
   - And PM Agent 返回澄清问题
   - And Web 页面显示问题供用户回答

2. **AC2: 用户回答处理**
   - Given 用户提交回答
   - When 系统接收回答
   - Then 继续澄清对话或确认需求明确
   - And 这是唯一的人工参与节点

**FRs covered:** FR10, FR11

## Dev Notes

### 实现说明

本 Story 的核心功能已在现有代码中实现：

1. **ClarificationManager** (`internal/workflow/clarification.go`)
   - 管理澄清轮次
   - 处理跳过逻辑
   - AI Agent 需求理解调用

2. **API Handlers** (`internal/server/handlers/api_handler.go`)
   - GET `/api/v1/:identifier/clarification` - 获取澄清状态
   - GET `/api/tasks/:identifier/clarification` - 获取澄清状态
   - POST `/api/tasks/:identifier/clarification/answer` - 提交回答
   - POST `/api/v1/:identifier/skip-clarification` - 跳过澄清

3. **新增 Planner** (`internal/harness/planner.go`)
   - 统一的规划器接口
   - 管理规划产出 (不可变)
   - 集成 AgentCaller 调用 BMAD agents
