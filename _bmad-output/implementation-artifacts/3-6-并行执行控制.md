# Story 3.6: 并行执行控制

Status: review

## Story

As a 系统架构师,
I want Generator 能正确控制并行执行,
so that 子任务可以安全地并行运行.

## Acceptance Criteria

1. **AC1: 并行执行机制**
   - Given Generator Phase 1 开始
   - When 启动 G1, G2, G3
   - Then 使用 goroutine 并行执行
   - And 使用 sync.WaitGroup 等待完成

2. **AC2: 失败处理**
   - Given 任一并行任务失败
   - When 检测到失败
   - Then 记录失败信息
   - And 等待其他任务完成

var wg sync.WaitGroup
var results [3]string
var errors [3]error

// G1: BDD 测试脚本

    defer wg.Done()
    results[0], errors[0] = g.generateBDDTestScript(ctx, taskID, plannerOutput)
}()

// G2, G3 类似...

// 等待所有任务完成
