# Story 8.3: 并发写入安全

Status: done

## Story

As a 系统架构师,
I want FileTracker 支持并发写入安全,
so that 多个子 agent 同时完成时不会产生文件冲突.

## Acceptance Criteria

1. **AC1: 调度器串行化写入**
   - Given 子 agent 完成任务
   - When 回调调度器
   Then 调度器更新内存状态
   And 调度器串行化写入文件
   And 无并发写文件问题

2. **AC2: 文件读取安全**
   - Given Web UI 或其他组件读取状态
   - When 并发读取请求
   - Then 文件读取不阻塞
   - And 返回一致的状态视图

3. **AC3: 服务崩溃恢复**
   - Given 服务崩溃后重启
   - When 恢复进行中任务
   - Then 从文件读取状态
   - And 恢复内存状态
   - And 继续执行

## Dev Notes

### 并发写入策略

**调度器作为唯一写入者**:
- 子 agent 完成后不直接写文件
- 子 agent 通过 channel/callback 通知调度器
- 调度器串行化处理更新请求
- 调度器统一写入文件

这是最简单的并发安全方案，避免文件锁和合并策略的复杂性。

### 文件读取策略

文件读取不需要锁：
- Go 文件读取是并发安全的
- 状态索引文件小，读取快
- 可在 FileClient 内部缓存状态，减少文件 I/O

### 崩溃恢复流程

```
服务启动 → 扫描 .sym/*/task.md → 找到 status=in-progress → 加载状态 → 恢复 Orchestrator
```
