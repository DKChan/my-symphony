Tracker=beads 时的任务流转

核心概念区分

两个独立的状态系统：
1. Beads Issue State（外部）：Todo → In Progress → Done（由 beads CLI 管理）
2. sbit Workflow Stage（内部）：clarification → bdd_review → architecture_review → implementation → verification（由
workflow engine 管理）

Beads CLI 调用详情

┌─────────────────────────────────────────────────────────────────────────────┐
│                           Beads CLI 命令调用                                 │

任务开始:
┌───────────────────────────────────────────────────────────────────────────┐
│ beads issue list --state Todo                                             │
│ beads issue list --state "In Progress"                                    │
│ --> 返回活跃状态的 Issue 列表                                               │
└───────────────────────────────────────────────────────────────────────────┘

获取任务详情:
┌───────────────────────────────────────────────────────────────────────────┐
│ beads issue show <identifier>                                             │
│ --> 返回 Issue 详情（标题、描述、状态等）                                    │
└───────────────────────────────────────────────────────────────────────────┘

状态同步:
┌───────────────────────────────────────────────────────────────────────────┐
│ beads issue update <identifier> --state "In Progress"                     │
│ beads issue update <identifier> --state "Done"                            │
│ --> 更新 Issue 在 Beads 中的状态                                           │
└───────────────────────────────────────────────────────────────────────────┘

阶段状态存储:
┌───────────────────────────────────────────────────────────────────────────┐
│ beads issue update <identifier> --stage <stage_json>                      │
│ --> 存储 sbit Workflow Stage 状态到 Beads Issue 的 Custom 字段              │
│ --> 用于崩溃恢复                                                            │
└───────────────────────────────────────────────────────────────────────────┘

│ --> 获取对话历史                                                            │
│                                                                            │
│ beads issue comment <identifier> --body <turn_json>                       │

关键点总结

1. Beads 只管理 Issue State（Todo/In Progress/Done）
2. sbit 管理 Workflow Stage（5个阶段 + 2个特殊阶段）
3. 阶段状态通过 --stage 存储到 Beads Custom 字段，用于崩溃恢复
4. 对话记录通过 comment 命令存储，用于多轮澄清
5. BDD/架构/验收内容通过 --field 存储，审核结果也通过 --field 更新
6. 子任务通过 --parent 和 --blocked-by 创建依赖关系
